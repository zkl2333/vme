## 提交消息建议 {{ date: '2023-04-06T00:00Z' }}

在最新的版本中，我添加了通过与 OpenAI 集成的提交消息和描述建议功能。Commit 会查看你所有的更改，并将它们输入到模型中，结合一些提示调优，返回一个提交消息，令人惊讶地准确描述了你的更改意图。

这也帮助我提醒自己在昨天结束工作时到底在做什么，当我回到电脑前并意识到自己没有提交任何工作时非常有用。

### <SparkleIcon /> 改进

- 添加了由 OpenAI 提供支持的提交消息和描述建议
- 修复了有时会导致你处于错误重写基状态的竞争条件问题
- 改进了活动项目的检测，尝试忽略由系统而不是用户触发的文件更改
- 修复了有时报告更改文件数量错误的 Bug

## 项目配置文件 {{ date: '2023-03-17T00:00Z' }}

我添加了创建每个项目的 `.commitrc` 文件的支持，用于覆盖特定项目的全局设置。选择使用 YAML 是因为我个人受够了在 JSON 中总是为键加引号，或者不小心留下了尾随逗号。

### <SparkleIcon /> 改进

- 添加了每个项目的 `.commitrc` 配置文件
- 在处理包含大量二进制文件的项目时提高了性能
- 修复了在删除最近活跃的分支后自动提交时可能导致 Commit 崩溃的 Bug

## Commit v0.1.0 {{ date: '2023-03-03T00:00Z' }}

Commit 是一个命令面板风格的 Git 客户端，你可以通过快捷键随时打开，轻松提交你的工作。它使用所有项目中文件的“最后修改”时间戳来自动知道你正在处理哪个项目，所以每次你拉起 UI 时，已经选中了正确的项目 —— 你只需输入提交消息，按下 <kbd>Cmd</kbd> + <kbd>Enter</kbd>，你的更改就被捕获了。

如果说我真的认为这工具有多大用，那就说谎了，不过我只是想找个借口学习 macOS 开发，而现在我们就在这里。至少它是开源的，所以即使这个应用本身完全浪费硬盘空间，或许你可以在代码中找到一些有趣的东西。
